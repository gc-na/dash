# [डेबियन] Debian Almquist Shell (dash) nice उपयोग: प्रक्रिया की प्राथमिकता को नियंत्रित करना

## Overview
`nice` कमांड का उपयोग प्रक्रियाओं की प्राथमिकता को नियंत्रित करने के लिए किया जाता है। यह कमांड एक प्रक्रिया को कम प्राथमिकता (nice value) देने की अनुमति देता है, जिससे अन्य प्रक्रियाएं अधिक प्राथमिकता के साथ चल सकें।

## Usage
कमांड का मूल सिंटैक्स इस प्रकार है:
```
nice [options] [arguments]
```

## Common Options
- `-n, --adjustment=N`: प्राथमिकता को समायोजित करने के लिए, जहाँ N एक संख्या है। सकारात्मक संख्या प्राथमिकता को घटाती है, जबकि नकारात्मक संख्या बढ़ाती है।
- `-h, --help`: मदद संदेश प्रदर्शित करता है।
- `--version`: `nice` कमांड का संस्करण दिखाता है।

## Common Examples
1. एक प्रक्रिया को कम प्राथमिकता के साथ चलाना:
   ```bash
   nice -n 10 command_name
   ```

2. एक स्क्रिप्ट को प्राथमिकता के साथ चलाना:
   ```bash
   nice -n 5 ./script.sh
   ```

3. प्राथमिकता को बिना किसी समायोजन के चलाना:
   ```bash
   nice command_name
   ```

4. एक प्रक्रिया को उच्च प्राथमिकता के साथ चलाना (नकारात्मक समायोजन):
   ```bash
   nice -n -5 command_name
   ```

## Tips
- `nice` का उपयोग करते समय ध्यान रखें कि नकारात्मक प्राथमिकता केवल सुपरयूजर द्वारा दी जा सकती है।
- यदि आप एक प्रक्रिया को चलाने के बाद उसकी प्राथमिकता बदलना चाहते हैं, तो `renice` कमांड का उपयोग करें।
- हमेशा प्राथमिकता को समायोजित करते समय सिस्टम की समग्र प्रदर्शन पर विचार करें।
