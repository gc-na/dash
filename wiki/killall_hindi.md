# [डेबियन] Debian Almquist Shell (dash) killall उपयोग: सभी प्रक्रियाओं को समाप्त करना

## Overview
`killall` कमांड का उपयोग एक ही नाम वाली सभी प्रक्रियाओं को समाप्त करने के लिए किया जाता है। यह विशेष रूप से तब उपयोगी होता है जब आप एक ही समय में चल रही कई प्रक्रियाओं को समाप्त करना चाहते हैं।

## Usage
`killall` कमांड का मूल सिंटैक्स इस प्रकार है:

```bash
killall [options] [arguments]
```

## Common Options
- `-u <user>`: केवल निर्दिष्ट उपयोगकर्ता द्वारा चलाए जा रहे प्रक्रियाओं को समाप्त करता है।
- `-i`: उपयोगकर्ता से पुष्टि मांगता है, इससे पहले कि वह प्रक्रियाओं को समाप्त करे।
- `-q`: कोई संदेश नहीं दिखाता है, केवल सफलतापूर्वक समाप्त की गई प्रक्रियाओं की संख्या दिखाता है।
- `-s <signal>`: एक विशेष सिग्नल का उपयोग करके प्रक्रियाओं को समाप्त करता है (डिफ़ॉल्ट सिग्नल `TERM` है)।

## Common Examples
यहाँ कुछ सामान्य उदाहरण दिए गए हैं:

1. सभी `firefox` प्रक्रियाओं को समाप्त करना:
   ```bash
   killall firefox
   ```

2. केवल एक विशेष उपयोगकर्ता द्वारा चलाए जा रहे `python` प्रक्रियाओं को समाप्त करना:
   ```bash
   killall -u username python
   ```

3. उपयोगकर्ता से पुष्टि मांगते हुए सभी `gedit` प्रक्रियाओं को समाप्त करना:
   ```bash
   killall -i gedit
   ```

4. एक विशेष सिग्नल का उपयोग करके सभी `ssh` प्रक्रियाओं को समाप्त करना:
   ```bash
   killall -s SIGKILL ssh
   ```

## Tips
- `killall` का उपयोग करते समय सावधानी बरतें, क्योंकि यह सभी समान नाम की प्रक्रियाओं को समाप्त कर सकता है, जो आपके कार्य को प्रभावित कर सकता है।
- यदि आप सुनिश्चित नहीं हैं कि कौन सी प्रक्रियाएँ समाप्त होंगी, तो `-i` विकल्प का उपयोग करें ताकि आप पुष्टि कर सकें।
- नियमित रूप से चलने वाली प्रक्रियाओं को समाप्त करने से पहले उनके कार्य को समझना महत्वपूर्ण है, ताकि आप अनजाने में महत्वपूर्ण प्रक्रियाएँ समाप्त न करें।
